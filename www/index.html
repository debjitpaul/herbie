Herbie

Automatically Improving Floating Point Accuracy

Correct-looking floating point code can produce incorrect results due to round-off error. Herbie is a project from the University of Washington to automatically fix these problems. Herbie automatically discovers and applies numerical analysis techniques, producing accurate programs without requiring any specialized knowledge from the user.

Status

Herbie works on numerical libraries, scientific publications, and mathematics textbooks. The graph below shows Herbie's accuracy improvements for 28 test cases drawn from Richard Hamming’s Numerical Methods for Scientists and Engineers are below. Herbie improves all results, many to almost-perfect precision.

Herbie has also created multiple patches to math.js and has helped our colleagues avoid rounding error in their machine learning algorithms. Herbie has been accepted for publication to PLDI'15 as a distinguished paper—read it!

Software using Herbie

  • The Herbie GHC Plugin by Mike Izbicki automatically runs Herbie on applicable expressions in a Haskell program. When run on the Stackage LTS-3.5 collection of popular Haskell libraries, this Herbie plugin finds 303 numerical expressions, of which 164 (54%) are made more accurate with Herbie.
  • The Herbie Rust Linter plugs into the Rust compiler to add warnings for numerically unstable expressions, and suggests Herbie's more accurate output as a hint.
